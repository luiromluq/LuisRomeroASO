# 04.03 - Caso practico 01 - Creando imagen Ubuntu con nano

## 1- Introducción

En este caso práctico vamos a crear y probar una imagen basada en Ubuntu que simplemente
incluirá el editor de texto de consola “nano”.

## 2- Preparando el Dockerfile y creando la imagen

En tu ordenador (o máquina virtual), crea una carpeta, por ejemplo:

    mkdir ubuntunano
    cd ubuntunano

<img width="463" height="99" alt="image" src="https://github.com/user-attachments/assets/6695c06b-2fc3-4cca-a5c5-14f309afa2de" />

Dentro de esa carpeta, crea un archivo llamado Dockerfile (sin extensión):

    nano Dockerfile

Crearemos el siguiente “Dockerfile”:

    #Imagen base ubuntu
    FROM ubuntu
    # Actualizamos lista de paquetes e instalamos nano (-y para no
    preguntar)
    # Las últimas líneas son para hacer la imagen más ligera
    RUN apt update && apt install -y nano && apt purge --auto-remove && apt
    clean && rm -rf /var/lib/apt/lists/*
    # Establecemos como comando por defecto de la imagen /bin/bash
    CMD /bin/bash

<img width="1204" height="237" alt="image" src="https://github.com/user-attachments/assets/8f527ecb-4fe7-4bf9-a4b2-b2cf9fb23a94" />

El funcionamiento del propio “Dockerfile” está definido por sus propios comentarios. Una vez
preparado, crearemos la imagen con:

    sudo docker build -t ubuntunano ./

Con esa línea indicamos que creamos la imagen “ubuntunano” basándose en el fichero
“Dockerfile” del directorio actual.

<img width="1197" height="383" alt="image" src="https://github.com/user-attachments/assets/b8df9260-238a-4cc4-8690-2a495497757e" />

## 3- Probando la imagen 

Con el siguiente comando, podremos crear un contenedor con esta imagen, acceder a una shell
dentro del contenedor y comprobar que el programa “nano” está instalado, ejecutando el editor
con “nano prueba.txt” o similar.

    docker run -it ubuntunano

<img width="702" height="56" alt="image" src="https://github.com/user-attachments/assets/f558f79d-f95e-40c7-ab43-00f4af630e92" />

Dentro del contenedor escribe:

    nano prueba.txt
