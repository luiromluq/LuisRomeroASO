# 3.1 El servicio de directorio

## Planificación de la fusión de directorios LDAP

Se plantea el caso de dos organizaciones, A y B, que se fusionan y disponen de estructuras de DIT similares. El objetivo es añadir la información del directorio B dentro del directorio A.

Una primera opción sería reescribir completamente el directorio A y eliminar los directorios anteriores, aunque esta solución resulta agresiva si solo es necesario realizar algunos cambios.

Renombrar directamente una unidad organizativa que contiene subentradas no es posible en LDAP. Por ejemplo, cambiar el nombre de una unidad organizativa que tiene entradas de tipo 

        inetOrgPerson 

provocará un error, ya que dichas entradas quedarían sin antepasado en el árbol del directorio.

La solución consiste en crear una nueva unidad organizativa con el nombre correcto, mover todas las entradas correspondientes a esta nueva ubicación y, posteriormente, eliminar la unidad organizativa obsoleta. Otra opción más sencilla es exportar el directorio a un archivo LDIF, vaciar el directorio, modificar el archivo exportado y volver a importarlo.

En el proceso de fusión se deben realizar los siguientes cambios:

- En los objetos de tipo "organizationalUnit":
  - Cambiar los atributos "dn", "dc" y "ou".
- En los objetos de tipo "inetOrgPerson":
  - Cambiar los atributos "dn" y "mail".
- En los objetos de tipo "device":
  - Cambiar los atributos "dn", "dc" y "ou".
    
## Creación de un archivo LDIF

A partir de los datos proporcionados se decide utilizar la clase "inetOrgPerson", ya que permite almacenar información personal y el correo electrónico.

La entrada se sitúa dentro de una unidad organizativa llamada "contactes", perteneciente al dominio "dc=empresa,dc=com".

    dn: cn=Luis Romero Luis,ou=contactes,dc=empresa,dc=com
    objectClass: inetOrgPerson
    cn: Luis Romero Luque
    givenName: Luis
    sn: Romero Luque
    homePhone: 623420265
    mobile: 623420265
    telephoneNumber: 925839013
    o: Lucho SL
    postalAddress: Carrer Jose Chabas, 3
    postalCode: 46019
    l: Valencia
    mail: lromeroluque@ies.com


## Archivo LDIF para modificar datos

La persona Luis Romero ha cambiado de dirección y deja de tener teléfono personal. Para realizar las modificaciones necesarias se utiliza un archivo LDIF con la directiva changetype: modify.

    dn: cn=Luis Romero Luque,ou=contactes,dc=empresa,dc=com
    changetype: modify
    delete: homePhone
    -
    replace: postalAddress
    postalAddress: Carrer Benito Villamarin, 4
    -
    replace: postalCode
    postalCode: 46012
    -
    replace: l
    l: Alboraya
